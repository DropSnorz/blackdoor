Gestion du Vibreur

Qu’est-ce qu’un Vibreur ?

Dans ce chapitre, nous allons voir comment travailler avec le vibreur d’un téléphone. Mais de quoi s’agit-il ? Sur la majorité des téléphones portables, il existe un petit mécanisme appelé vibreur, qui permet de notifier discrètement à son propriétaire un événement. Par exemple, le vibreur peut-être utilisé lors de la réception d’un appel, d’un message, d’une notification émise par une application ou encore lorsqu’on éteint le téléphone. Techniquement, Il s'agit le plus souvent d'une masse excentrée entraînée en rotation par un micro-moteur électrique.

Info - Dans d'autres cas, afin d'obtenir une forme de vibration plus fine du contrôleur de jeu, des actionneurs piézoélectriques sont sollicités. C'est le cas des manettes du visiocasque HTC vive.

Le système de vibration n’est pas exclusivement réservé aux téléphones mobiles, puisqu’il est également utilisé sur les consoles de jeu portables, ainsi que sur les manettes de jeu des consoles de table, pour fournir des sensations aux joueurs. Par exemple, il est fréquent d’utiliser le vibreur pour faire vibrer les manettes dans les jeux vidéos, lorsque le personnage prends un coup, ou encore lorsqu’une interaction avec un autre personnage à lieu.

Utiliser le vibreur sur un téléphone Android

Pour utiliser la vibration, Android nous met à disposition une classe portant le nom de Vibrator. C’est en manipulant des objets provenant de cette classe que l’on va pouvoir faire vibrer le téléphone de l’utilisateur. Pour obtenir une instance de cette classe, nous allons utiliser la fonction getSystemService(Class), avec un VIBRATOR_SERVICE comme argument.

Attention - Il est nécessaire d'inclure android.permission.VIBRATE en tant que dépendance pour utiliser la classe Vibrator

Il existe de nombreuses fonctionnalités offertes par cette classe. Il est important de noter que si l'utilisateur quitte l'application, la vibration s'arrète . Voyons ensemble une liste non-exhaustive des fonctions disponibles

Type Fonction
abstract void cancel()
Desactive la vibration
abstract boolean hasAmplitudeControl()
Verifie si la vibration est controlée par amplitude
abstract boolean hasVibrator()
Verifie si l'appareil est équipé d'un vibreur
void vibrate(long milliseconds)
Déclenche une vibration d'une durée égale au paramètre millisecondes
Attention - Beaucoup de fonctions sont disponibles, dont certaines dépréciées. Pour plus d'informations, consultez l'API Android

Si il ne fallait retenir qu'une seule fonction, c'est sans aucun doute vibrate(long milliseconds). Cette fonction va nous permettre de lancer la vibration.

A vous de jouez !

Plaçons nous dans un contexte précis. Supposons que nous souhaitons lancer une vibration sur le mobile de l'utilisateur. Il faut, dans l'ordre :

  • 1- Instancier un objet Vibrator
  • 2- Tester la présence ou l'absence du vibreur sur le mobile
  • 3- Si le vibreur est présent, alors on lance la vibration (pour l'exemple, supposons une vibration d'une seconde)
  • 4- Sinon, on ne fait rien
